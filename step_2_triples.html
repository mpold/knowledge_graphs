Step 2 — triples pipeline (gpu_bundle)

Order of execution, inputs and outputs of the Python scripts in lung/gpu_bundle/ — the Kaggle/GPU bundle that takes the NER corpus handed over by step 1 (experimental_ner/) and turns it into normalized, learned-scored relation triples. One orchestrator, gpu.py, runs a fixed 16-step chain in dependency order; each step is a standalone script run as a subprocess in a single writable working directory, and the final step zips that directory into one downloadable archive.

You normally run one command — python gpu.py — and it executes all 16 steps for you, in the order below, in one staged working directory (/kaggle/working/ on Kaggle, or in place locally). It is not a loose set of scripts you run by hand: gpu.py copies the scripts, symlinks the read-only experimental_ner/ corpus, makes a writable copy of databases/, creates the output dirs (GENETIC/ DISEASE/ CHEMICAL/ TRIPLES/, plus sentences/ and summaries/), and sets RE_MODEL_PPI for the learned RE step. Steps 1, 2 and 15 use CUDA; steps 1, 2 and 14 reach the network. Steps 13 and 14 are optional — if they fail the orchestrator warns and continues to the critical RE step. The last step (zip_work.py) bundles the whole working directory into kaggle_working.zip so the entire run is a single download. Run a subset with --steps a,b,c and preview the plan with --list.

Steps 3–11 all read the same sentences/*.json and only touch their own type's library dir, so within a type they are ordered by dependency: roman.py (step 3) builds the GENETIC libraries that greek.py/keys_values.py/controls.py extend or report on; controls.py (6) re-annotates those libraries in place, so it runs after roman.py/greek.py rewrite them. triples.py (12) reads all three finished libraries; relationships.py (13) reads triples.py's output; pub_years.py (14) reads relationships.py's genetic_genetic.json. The learned RE step (15) depends only on step 1's model and step 2's sentences, so it is independent of the 13–14 branch.

3. How to run

  1. 1python gpu.py --list — preview the plan (roots, input count, accelerator, the 16 steps).
  2. 2python gpu.py — run all 16 steps end-to-end. On Kaggle: enable the GPU accelerator + internet, and !pip install -q 'datasets<4' first (for step 1's BigBIO download).
  3. 3python gpu.py --steps triples,relation_extraction — run a subset (e.g. re-derive triples from existing sentences/ and libraries).

Individual scripts can still be run directly (python triples.py, etc.) — each resolves its paths relative to its own file — but gpu.py is the intended entry point because it stages inputs, orders the steps, and wires up RE_MODEL_PPI.

Step 1 (run_re_pipeline.py) needs internet + datasets<4 to download BigBIO bioinfer; step 2 (sentences.py) pulls the BioBERT NER models from Hugging Face on first use; step 14 (pub_years.py) calls NCBI. Steps 1, 2 and 15 run on CUDA (they fall back to CPU — slow — if no GPU is visible). Upload ppi-biobert-re/ and skip step 1, or upload sentences/ and skip step 2, to reuse prior results.

4. Notes

  • One orchestrator, one working dir. gpu.py stages the scripts, experimental_ner/ (symlinked), a writable databases/ copy, and the output dirs into a single run root, then runs each step there as a subprocess with the same environment.
  • The hub file is sentences/*.json. Step 2 produces it from the XML corpus; steps 3–12 and 15 all read it. Three parallel normalization branches (GENETIC / DISEASE / CHEMICAL) fan out from it and reconverge in triples.py.
  • Two triple producers. triples.py (12) is rule-based (relation-cue text between consecutive entities); relation_extraction.py (15) is the trained BioBERT model. Both write into TRIPLES/; step 15's scores are the learned counterpart to step 12's candidates.
  • Normalization is layered. triples.py writes the base triples then three progressively normalized files: +HGNC, +MONDO, +ChEBI — the last (triples_GENETIC_DISEASE_CHEMICAL_normalized.json) is the fully ID-normalized set.
  • In-place annotators. controls.py (6), phenotypes.py (8) and nonchemical.py (10) add yes/no flags onto existing library JSON rather than producing new corpora — so they must run after the libraries they annotate are built.
  • Hand-off from step 1. The input experimental_ner/ is exactly what the publications pipeline (step_1_publications.html) assembled; this bundle is the "step 2" referenced there.
  • Single-download bundle. The last step, zip_work.py (16), packs the whole working directory into kaggle_working.zip — so the entire run (model, sentences, normalization libraries, TRIPLES/ and the HTML summaries) comes off Kaggle as one file. It runs after every other step, and builds the archive in a temp dir so the growing zip can't include itself.
