Requirements — running the full pipeline

What you need to run the three stages of the project end to end: step 1 — publications (step_1_publications.html), step 2 — triples / GPU bundle (step_2_triples.html) and step 3 — graph (step_3_graph.html). The three stages run in different environments: step 1 is a local, network-heavy pipeline (+ Docker for GROBID), step 2 is a GPU job best run on Kaggle, and step 3 is a lightweight local script.

The stages hand off by files, so each has its own prerequisites. Step 1 produces the NER corpus gpu_bundle/experimental_ner/; step 2 (the GPU bundle, orchestrated by gpu.py) turns it into triples and packs the whole run into kaggle_working.zip; step 3 (high_confidence.py) reads that unzipped run and draws the graph. Common to all: Python 3 and the ability to reach the network on the stages that fetch data/models.

1. Common requirements (all stages)

PythonPython 3 (3.9+ recommended). All scripts are launched as python <script>.py and resolve their paths relative to their own location, so they can be run from any working directory.
NetworkInternet access on the stages that fetch data (step 1 throughout; step 2 for corpus/model downloads and the year lookup; step 3 only for the optional graph library).
DiskRoom for the corpus, models and the packaged run (the step-2 output tree and its kaggle_working.zip can be multi-GB).
OrderStrictly sequential: 1 → 2 → 3. Each stage needs the file outputs of the one before it.

2. Step 1 — publications local internet Docker

Seven scripts in the project root, run in order (pubmed_query.py → high_impact_xml.py → xml_structure.py → ncbi_pdf.py → grobid_xml.py → named_entity_xml.py → pre_ner_xml_structure.py). Not run automatically — run each yourself.

Python packagesrequests (used by high_impact_xml.py, ncbi_pdf.py, grobid_xml.py). The rest use only the standard library (urllib, etc.). Install: pip install requests.
InputA PubMed query read strictly from STDIN by pubmed_query.py (e.g. echo "pancreatic cancer" | python pubmed_query.py).
External servicesNCBI E-utilities (esearch/esummary/eFetch, ID Converter), OpenAlex, CrossRef, PMC — reached over the network for PMIDs, metadata, impact factors and PDFs.
Docker + GROBIDOnly grobid_xml.py (step 5): a running GROBID service on :8070. It can auto-launch Docker Desktop + the GROBID container. Skippable when every article already has JATS full text. Docker is not needed for any other script.
Env vars (optional)NCBI_API_KEY (recommended — lifts the 3 req/s rate limit); JIF_TABLE, YEAR_LO/YEAR_HI, TIME_BUDGET, IF_THRESHOLD, PERCENTILE, RETRY_FAILED, DRY_RUN; and the GROBID_* family (server, threads, timeouts, autolaunch, image/container) for step 5.
ProducesThe NER corpus gpu_bundle/experimental_ner/PMC*.xml (+ per-script HTML summaries in summaries/) — the input to step 2.

3. Step 2 — triples / GPU bundle Kaggle GPU / CUDA internet

One orchestrator, gpu.py, runs a 16-step chain in a single working directory (/kaggle/working/ on Kaggle, or in place locally). Best run on Kaggle with a GPU accelerator.

HardwareGPU with CUDA — steps 1, 2 and 15 (model training, BioBERT NER, learned relation extraction) use CUDA. They fall back to CPU but are then very slow. On Kaggle: Settings → Accelerator → GPU.
Python packagestorch + transformers (training, NER, RE), numpy, and lxml (step 2 XML parsing) — all preinstalled on Kaggle GPU images. Plus datasets<4 for the BigBIO download, pinned in a setup cell: !pip install -q 'datasets<4' (BigBIO ships as loading scripts dropped in datasets 4.0). Steps 3–14 are pure standard library.
Network / modelsInternet ON: step 1 downloads the BigBIO bioinfer corpus (HF datasets); step 2 pulls the BioBERT models (dmis-lab/biobert-v1.1, alvaroalon2/biobert_{diseases,genetic,chemical}_ner) from Hugging Face on first use; step 14 (optional) calls NCBI for publication years.
Data inputsexperimental_ner/ (from step 1) and the ontology databases under databases/: hgnc_complete_set_2026-05-01.json (HGNC), mondo-clingen.json (MONDO), chebi.json (ChEBI). Upload these with the scripts. (pmc_years.json is generated by step 14; upload it only to run the year filter offline.)
Scripts to uploadThe 16 step scripts + bigbio_to_re.py, train_re.py, calibration.py. ppi-biobert-re/ (model) and sentences/ are generated by steps 1 and 2 — no need to upload unless you skip those steps.
Runpython gpu.py --list to preview, then python gpu.py for all 16 steps. Steps 13–14 are optional (warn & continue on failure).
ProducesTRIPLES/ (incl. the scored/normalized triples) and, from the final step, kaggle_working.zip — the whole run bundled for download.

4. Step 3 — graph local internet (optional)

One standalone script, high_confidence.py, run on your local computer against the unzipped step-2 output.

NB! kaggle_working.zip (produced by step 2) must be downloaded to the local computer and unzipped before high_confidence.py is triggered — it reads its inputs from that unzipped run directory (the data root).
Python packagesNone beyond the standard library (argparse, json, collections, re, html, shutil, urllib). No torch/transformers needed — this stage does no model inference.
Data inputsThe unzipped kaggle_working/ tree (the data root; override with --data-root). Required: TRIPLES/triples_re_GENETIC_DISEASE_CHEMICAL_normalized.json and TRIPLES/triples.json. Optional (enrich the graph): databases/pmc_years.json, CHEMICAL/chemical_to_target.json, DISEASE/disease.json, DISEASE/disease_ambiguous.json, sentences/, and summaries/pubmed_query.html (the query legend line).
Network (optional)Fetches vis-network from a CDN to inline into the graph HTML. If offline, the HTML falls back to a CDN link, or run with --no-graph to skip it entirely.
Runpython high_confidence.py (options: --data-root, --score, --thresholds, --no-graph).
ProducesTRIPLES/high_confidence_G_D_C.json, the interactive summaries/high_confidence.html, and a copy of the graph in the root dir named after the current directory plus today's date (e.g. lung_large_2026_07_19.html).

5. End-to-end checklist

  1. 1Step 1 (local): pip install requests; have Docker Desktop available for GROBID; optionally set NCBI_API_KEY. Run the seven scripts in order, starting with echo "your query" | python pubmed_query.py.
  2. 2Step 2 (Kaggle GPU): upload the scripts + experimental_ner/ + the three ontology JSONs; enable GPU + Internet; !pip install -q 'datasets<4'; run python gpu.py. Download the resulting kaggle_working.zip.
  3. 3Step 3 (local): unzip kaggle_working.zip, then run python high_confidence.py (needs only Python 3; internet optional for the graph).
The heaviest prerequisites are stage-specific: Docker + GROBID for step 1's PDF-to-TEI conversion, and a CUDA GPU + torch/transformers/datasets for step 2's training and inference. Step 3 deliberately needs neither — just Python 3 and the unzipped run.
